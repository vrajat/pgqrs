# Agent Docs

Shared project instructions live in this directory.

## Load Order

1. `process.md` for shared workflow, testing, and quality expectations.
2. `context/technical-domain.md` and `context/living-notes.md` for current repo context.
3. `personas/*.md` when a task benefits from role-specific guidance.

## Structure

- `process.md` - shared project process and conventions
- `context/` - current project context worth loading during planning
- `personas/` - planner, builder, tester, reviewer guidance

## Wrappers

Thin model-specific entry points remain at the repo root:

- `../AGENTS.md` - Codex/GPT wrapper
